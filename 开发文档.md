# 开发文档

本文档说明如何在本地开发和用 Docker Compose 构建运行当前源码版本。

## 1. 基础环境

推荐环境：

- Windows 10/11 + Docker Desktop
- Docker Compose v2
- WSL2
- Git
- Go 1.22+，仅在本机直接运行后端时需要
- Bun，主要用于前端本地开发

检查命令：

```powershell
docker --version
docker compose version
wsl --version
wsl -l -v
git --version
```

如果 Docker Desktop 提示 WSL 异常，可以先执行：

```powershell
wsl --update --web-download
wsl --set-default-version 2
wsl --shutdown
```

然后重启 Docker Desktop。

## 2. 克隆项目

```powershell
git clone https://github.com/maker857/new-api.git
cd new-api
```

```powershell
cd D:\newapi
git fetch origin
```

## 3. 本地环境变量

项目根目录创建 `.env`。`.env` 不要提交到 Git。

开发示例：

```dotenv
POSTGRES_PASSWORD=123456
REDIS_PASSWORD=123456
SESSION_SECRET=change-me-session-secret
CRYPTO_SECRET=change-me-crypto-secret
TZ=Asia/Shanghai
```

说明：

- `POSTGRES_PASSWORD`：本地 PostgreSQL 密码。
- `REDIS_PASSWORD`：本地 Redis 密码。
- `SESSION_SECRET`：会话密钥，生产环境必须换成高强度随机值。
- `CRYPTO_SECRET`：加密密钥，生产环境必须换成高强度随机值。
- `TZ`：容器时区。

## 4. 本地 Docker 覆盖配置

原版项目直接 `docker compose up -d` 会使用远程镜像。开发本项目源码时，需要让 Compose 从当前目录构建镜像，因此建议创建本地覆盖文件 `docker-compose.local.yml`。

`docker-compose.local.yml` 不建议提交，它属于每台机器自己的本地配置。

模板：

```yaml
services:
  new-api:
    image: new-api-local:dev
    build:
      context: .
    environment:
      - SQL_DSN=postgresql://root:${POSTGRES_PASSWORD}@postgres:5432/new-api
      - REDIS_CONN_STRING=redis://:${REDIS_PASSWORD}@redis:6379
      - SESSION_SECRET=${SESSION_SECRET}
      - CRYPTO_SECRET=${CRYPTO_SECRET}
      - TZ=${TZ}

  redis:
    command: ["redis-server", "--requirepass", "${REDIS_PASSWORD}"]

  postgres:
    environment:
      POSTGRES_PASSWORD: ${POSTGRES_PASSWORD}
```

## 5. 构建并启动本地源码版

```powershell
docker compose -f docker-compose.yml -f docker-compose.local.yml up -d --build
```

访问：

```text
http://localhost:3000
```

查看状态：

```powershell
docker compose -f docker-compose.yml -f docker-compose.local.yml ps
```

正常情况下应看到 `new-api` 为 `healthy`。

## 6. 前端改动如何生效

当前 Dockerfile 会把 `web/default` 和 `web/classic` 前端构建产物嵌入 Go 二进制。

因此，修改前端源码后，仅重启容器不够，需要重新构建镜像：

```powershell
docker compose -f docker-compose.yml -f docker-compose.local.yml up -d --build new-api
```

如果浏览器仍显示旧页面，强刷新：

```text
Ctrl + F5
```

也可以检查页面是否加载了新的静态包：

```powershell
$html = (Invoke-WebRequest -UseBasicParsing "http://localhost:3000/" -Headers @{ "Cache-Control" = "no-cache" }).Content
$html
```

## 7. 后端改动如何生效

后端 Go 代码也会编译进镜像。修改后端源码后执行：

```powershell
docker compose -f docker-compose.yml -f docker-compose.local.yml up -d --build new-api
```

查看后端日志：

```powershell
docker logs --tail 100 new-api
docker logs -f new-api
```

健康检查：

```powershell
Invoke-RestMethod -Uri http://localhost:3000/api/status | ConvertTo-Json -Depth 5
```

返回中包含 `"success": true` 表示服务正常。

## 8. 常用 Docker 命令

启动：

```powershell
docker compose -f docker-compose.yml -f docker-compose.local.yml up -d
```

重新构建并启动：

```powershell
docker compose -f docker-compose.yml -f docker-compose.local.yml up -d --build
```

重启服务：

```powershell
docker compose -f docker-compose.yml -f docker-compose.local.yml restart new-api
```

停止：

```powershell
docker compose -f docker-compose.yml -f docker-compose.local.yml down
```

查看日志：

```powershell
docker compose -f docker-compose.yml -f docker-compose.local.yml logs -f new-api
docker compose -f docker-compose.yml -f docker-compose.local.yml logs --tail=100 postgres
docker compose -f docker-compose.yml -f docker-compose.local.yml logs --tail=100 redis
```

清空本地数据重新开始。注意：这会删除数据库卷和本地数据：

```powershell
docker compose -f docker-compose.yml -f docker-compose.local.yml down -v
Remove-Item -Recurse -Force .\data, .\logs -ErrorAction SilentlyContinue
docker compose -f docker-compose.yml -f docker-compose.local.yml up -d --build
```

## 9. 请求日志开发说明

本分支增加了请求日志文件保存能力。

相关配置：

- 渠道配置中有请求日志保存开关。只有开启该渠道开关，才保存请求日志内容。
- 系统设置中可以配置运行日志最大体积和请求日志最大体积。
- 最大体积为 `0` 表示不限制。

本地 Docker 默认运行日志目录：

```text
D:\newapi\logs
```

请求日志会按渠道名称创建目录，格式类似：

```text
logs\<渠道名称>\YYYY-MM-DD\<time>-<request-id>.log
```

运行日志和请求日志超过限制后，会优先删除最旧文件，避免磁盘被无限占满。

## 10. 前端开发模式

如果只开发 `web/default` 前端，可以使用开发 Compose：

```powershell
docker compose -f docker-compose.dev.yml up -d
cd web/default
bun install
bun run dev
```

开发服务器地址通常是：

```text
http://localhost:3001
```

如果修改了后端 Go 代码，需要重新构建后端容器：

```powershell
docker compose -f docker-compose.dev.yml up -d --build new-api
```
